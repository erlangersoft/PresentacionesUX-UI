# 3.2. Diseño Web y Maquetación

## Introducción

La maquetación web es el proceso de organizar elementos visuales en una página para crear jerarquía, guiar la atención y facilitar la comprensión del contenido. Una maquetación efectiva combina principios de diseño visual con conocimiento técnico de tecnologías web.

**Objetivos de la Maquetación:**
- Organizar contenido de manera lógica
- Crear jerarquía visual clara
- Facilitar navegación y escaneo
- Adaptarse a diferentes dispositivos
- Optimizar legibilidad y usabilidad

---

## 3.2.1. Principios de Estructura y Flujo de Contenido

### Flujo de Lectura Natural

En culturas occidentales, el flujo de lectura sigue patrones predecibles que deben considerarse en la maquetación.

### Patrón F (F-Pattern)

Descubierto por Nielsen Norman Group mediante eye-tracking.

**Comportamiento del usuario:**

```
F═══════════════════
║
F════════════
║
F══════
║
║

Usuarios leen:
1. Horizontalmente la parte superior (barra del F)
2. Hacia abajo del lado izquierdo (tallo del F)
3. Horizontalmente sección media (barra inferior del F)
4. Continúan bajando por izquierda
```

**Implicaciones de diseño:**
- ✓ Información más importante en las dos primeras líneas
- ✓ Primeras palabras de títulos y párrafos son críticas
- ✓ Lado izquierdo es "hot zone"
- ✓ Contenido debe ser escaneable

**Aplicación en EduConnect:**

```html
<!-- Optimizado para F-Pattern -->
<article class="course-detail">
  <!-- Primera línea horizontal: Información crítica -->
  <h1>Fundamentos de Python para Data Science</h1>
  <div class="meta">
    <span class="rating">★★★★★ 4.8</span>
    <span class="students">12,458 estudiantes</span>
    <span class="level">Intermedio</span>
  </div>
  
  <!-- Contenido escaneable por lado izquierdo -->
  <section class="description">
    <h2>Lo que aprenderás:</h2>
    <ul>
      <li><strong>NumPy</strong> para análisis numérico</li>
      <li><strong>Pandas</strong> para manipulación de datos</li>
      <li><strong>Matplotlib</strong> para visualización</li>
    </ul>
  </section>
</article>
```

### Patrón Z (Z-Pattern)

Común en páginas con menos texto, más visuales.

```
1══════════════════> 2
  ╲
   ╲
    ╲
     ╲
      ╲
       3══════════> 4

Usuarios escanean:
1. Esquina superior izquierda
2. Superior derecha
3. Diagonal hacia inferior izquierda
4. Inferior derecha
```

**Aplicación:** Landing pages, hero sections

```html
<!-- Optimizado para Z-Pattern -->
<section class="hero">
  <div class="hero-content">
    <!-- 1. Superior izquierda: Logo/Marca -->
    <img src="logo.svg" alt="EduConnect" class="logo">
    
    <!-- 2. Superior derecha: CTA secundario -->
    <button class="btn-secondary">Iniciar Sesión</button>
    
    <!-- 3. Centro-izquierda: Mensaje principal -->
    <h1>Aprende Skills del Futuro</h1>
    <p>Cursos online de tecnología con certificación</p>
    
    <!-- 4. Inferior derecha: CTA principal -->
    <button class="btn-primary">Explorar Cursos →</button>
  </div>
</section>
```

### Patrón de Capa (Layer Cake Pattern)

Secciones horizontales completas que se apilan verticalmente.

```
┌─────────────────────────────────┐
│  Hero Section (full width)      │
├─────────────────────────────────┤
│  Features Section               │
├─────────────────────────────────┤
│  Testimonials Section           │
├─────────────────────────────────┤
│  CTA Section                    │
├─────────────────────────────────┤
│  Footer                         │
└─────────────────────────────────┘
```

**Ventajas:**
- ✓ Narrativa clara y secuencial
- ✓ Fácil de hacer responsive
- ✓ Guía natural hacia abajo
- ✓ Separación clara de contenido

### Jerarquía Visual

**Principios de jerarquía en maquetación:**

**1. Size (Tamaño)**
```html
<h1 class="text-5xl">Más importante</h1>      <!-- 48px -->
<h2 class="text-3xl">Importante</h2>          <!-- 30px -->
<p class="text-base">Normal</p>               <!-- 16px -->
<small class="text-sm">Menos importante</small> <!-- 14px -->
```

**2. Weight (Peso)**
```html
<p class="font-bold">Énfasis fuerte</p>        <!-- 700 -->
<p class="font-semibold">Énfasis medio</p>     <!-- 600 -->
<p class="font-normal">Normal</p>              <!-- 400 -->
<p class="font-light">Secundario</p>           <!-- 300 -->
```

**3. Proximity (Proximidad)**
```html
<article class="space-y-6">
  <!-- Grupo 1: Relacionado (poco espacio) -->
  <div class="space-y-2">
    <h2>Título del Curso</h2>
    <p class="subtitle">Subtítulo descriptivo</p>
  </div>
  
  <!-- Separación entre grupos (más espacio) -->
  
  <!-- Grupo 2: Otro contenido relacionado -->
  <div class="space-y-2">
    <h3>Requisitos</h3>
    <ul>...</ul>
  </div>
</article>
```

**4. Color/Contrast**
```css
.primary-content {
  color: var(--gray-900);  /* Alto contraste - Más importante */
}

.secondary-content {
  color: var(--gray-600);  /* Contraste medio - Importante */
}

.tertiary-content {
  color: var(--gray-500);  /* Bajo contraste - Menos importante */
}
```

## 3.2.2. Sistemas de Grillas (Columnas, Gutters, Margins)

Las grillas son estructuras invisibles que organizan contenido, crean alineación y establecen ritmo visual.

### Anatomía de una Grilla

```
┌────────────────────────────────────────────────────┐
│ ← Margin →                         ← Margin →      │
│           ┌───┬─┬───┬─┬───┬─┬───┬─┬───┐            │
│           │Col│G│Col│G│Col│G│Col│G│Col│            │
│           │   │ │   │ │   │ │   │ │   │            │
│           └───┴─┴───┴─┴───┴─┴───┴─┴───┘            │
│                  ↑                                 │
│                Gutter                              │
└────────────────────────────────────────────────────┘

Componentes:
- Columns: Unidades verticales de contenido
- Gutters: Espacios entre columnas
- Margins: Espacios laterales exteriores
```

### Grilla de 12 Columnas (Estándar Web)

12 columnas es estándar porque 12 es divisible por 2, 3, 4 y 6, ofreciendo máxima flexibilidad.

```
Layouts posibles con 12 columnas:

2 columnas:
[══════6══════][══════6══════]

3 columnas:
[════4════][════4════][════4════]

4 columnas:
[═══3═══][═══3═══][═══3═══][═══3═══]

Asimétricos:
[════8════════][════4════]  (Contenido principal + Sidebar)
[═══3═══][══════6══════][═══3═══]  (Sidebar + Main + Sidebar)
```

### Especificaciones de Grilla para EduConnect

```css
/* Desktop (1200px container) */
.grid-container {
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 24px;  /* Margins */
}

.grid {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  gap: 24px;  /* Gutters */
}

/* Columna ocupa 6 de 12 (50%) */
.col-6 {
  grid-column: span 6;
}

/* Columna ocupa 4 de 12 (33.33%) */
.col-4 {
  grid-column: span 4;
}

/* Columna ocupa 8 de 12 (66.66%) */
.col-8 {
  grid-column: span 8;
}
```

### Grillas Responsive

```css
/* Mobile: 4 columnas */
@media (max-width: 767px) {
  .grid {
    grid-template-columns: repeat(4, 1fr);
    gap: 16px;
    padding: 0 16px;
  }
}

/* Tablet: 8 columnas */
@media (min-width: 768px) and (max-width: 1023px) {
  .grid {
    grid-template-columns: repeat(8, 1fr);
    gap: 20px;
    padding: 0 20px;
  }
}

/* Desktop: 12 columnas */
@media (min-width: 1024px) {
  .grid {
    grid-template-columns: repeat(12, 1fr);
    gap: 24px;
    padding: 0 24px;
  }
}
```

### Breakouts (Salirse de la Grilla)

A veces necesitas que elementos excedan la grilla regular.

```css
.grid-container {
  display: grid;
  grid-template-columns: 
    [full-start] 1fr
    [content-start] minmax(0, 1200px) [content-end]
    1fr [full-end];
  gap: 24px;
}

/* Contenido normal dentro de grilla */
.content {
  grid-column: content;
}

/* Elemento que ocupa ancho completo */
.full-bleed {
  grid-column: full;
}

/* Ejemplo: Imagen hero full-width */
.hero-image {
  grid-column: full;
  width: 100%;
}
```

## 3.2.3. Layouts: Flexbox y Grid

CSS Grid y Flexbox son las dos tecnologías modernas para crear layouts. Cada una tiene fortalezas específicas.

### Flexbox: Para Layouts Unidimensionales

Flexbox organiza elementos en una dirección (fila o columna).

**Cuándo usar Flexbox:**
- ✓ Navegación horizontal
- ✓ Centrar elementos
- ✓ Distribuir espacio entre items
- ✓ Alinear items en un eje
- ✓ Componentes pequeños (cards, buttons groups)

**Propiedades del Container:**

```css
.flex-container {
  display: flex;
  
  /* Dirección */
  flex-direction: row | column | row-reverse | column-reverse;
  
  /* Wrap (envolver a nueva línea) */
  flex-wrap: nowrap | wrap | wrap-reverse;
  
  /* Alineación horizontal (main axis) */
  justify-content: flex-start | center | flex-end | space-between | space-around | space-evenly;
  
  /* Alineación vertical (cross axis) */
  align-items: stretch | flex-start | center | flex-end | baseline;
  
  /* Espacio entre items */
  gap: 16px;
}
```

**Propiedades de Items:**

```css
.flex-item {
  /* Crecimiento (ocupar espacio disponible) */
  flex-grow: 1;
  
  /* Encogimiento (reducirse si es necesario) */
  flex-shrink: 1;
  
  /* Tamaño base */
  flex-basis: 200px;
  
  /* Shorthand: grow shrink basis */
  flex: 1 1 200px;
  
  /* Alineación individual */
  align-self: center;  /* Sobrescribe align-items del container */
}
```

### Ejemplos Prácticos con Flexbox

**1. Navegación Horizontal:**

```html
<nav class="main-nav">
  <a href="/">Inicio</a>
  <a href="/cursos">Cursos</a>
  <a href="/about">Acerca de</a>
  <a href="/perfil">Mi Perfil</a>
</nav>
```

```css
.main-nav {
  display: flex;
  gap: 24px;
  align-items: center;
}

.main-nav a {
  padding: 8px 16px;
  text-decoration: none;
  color: var(--gray-700);
}
```

**2. Card con Header y Footer:**

```html
<div class="card">
  <div class="card-header">Título</div>
  <div class="card-body">
    Contenido que puede variar en altura...
  </div>
  <div class="card-footer">
    <button>Acción</button>
  </div>
</div>
```

```css
.card {
  display: flex;
  flex-direction: column;
  height: 100%;
}

.card-header,
.card-footer {
  flex-shrink: 0;  /* No encoger */
}

.card-body {
  flex-grow: 1;  /* Ocupar espacio disponible */
}
```

**3. Centrar Elemento:**

```css
.center-container {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 400px;
}
```

**4. Grid de Cards Responsive con Flexbox:**

```html
<div class="card-grid">
  <div class="card">...</div>
  <div class="card">...</div>
  <div class="card">...</div>
</div>
```

```css
.card-grid {
  display: flex;
  flex-wrap: wrap;
  gap: 24px;
}

.card {
  flex: 1 1 300px;  
  /* grow: 1, shrink: 1, basis: 300px */
  /* Crea grid responsive automático */
}
```

### CSS Grid: Para Layouts Bidimensionales

Grid organiza elementos en filas Y columnas simultáneamente.

**Cuándo usar Grid:**
- ✓ Layouts de página completa
- ✓ Grids complejos
- ✓ Overlapping content
- ✓ Alineación precisa en 2 dimensiones

**Propiedades del Container:**

```css
.grid-container {
  display: grid;
  
  /* Definir columnas */
  grid-template-columns: 200px 1fr 200px;
  grid-template-columns: repeat(3, 1fr);
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  
  /* Definir filas */
  grid-template-rows: auto 1fr auto;
  
  /* Áreas nombradas */
  grid-template-areas:
    "header header header"
    "sidebar main main"
    "footer footer footer";
  
  /* Espacios */
  gap: 24px;
  row-gap: 24px;
  column-gap: 24px;
}
```

**Propiedades de Items:**

```css
.grid-item {
  /* Colocación por líneas */
  grid-column-start: 1;
  grid-column-end: 3;
  grid-column: 1 / 3;  /* Shorthand */
  grid-column: span 2;  /* Ocupar 2 columnas */
  
  /* Por área */
  grid-area: header;
  
  /* Alineación */
  justify-self: start | center | end | stretch;  /* Horizontal */
  align-self: start | center | end | stretch;    /* Vertical */
}
```

### Ejemplos Prácticos con Grid

**1. Layout de Página Clásico:**

```html
<div class="page-layout">
  <header class="header">Header</header>
  <aside class="sidebar">Sidebar</aside>
  <main class="main">Main Content</main>
  <footer class="footer">Footer</footer>
</div>
```

```css
.page-layout {
  display: grid;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
  grid-template-columns: 250px 1fr;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
  gap: 0;
}

.header { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main { grid-area: main; }
.footer { grid-area: footer; }
```

**2. Grid de Cursos (Catálogo EduConnect):**

```html
<div class="course-grid">
  <article class="course-card">...</article>
  <article class="course-card">...</article>
  <article class="course-card">...</article>
  <!-- ... más cards -->
</div>
```

```css
.course-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
  gap: 24px;
}

/* auto-fill: Crea tantas columnas como quepan */
/* minmax(300px, 1fr): Mínimo 300px, máximo 1 fracción */
/* Resultado: Grid totalmente responsive sin media queries */
```

**3. Hero Section con Overlay:**

```html
<section class="hero">
  <img src="hero-bg.jpg" alt="" class="hero-image">
  <div class="hero-content">
    <h1>Título Hero</h1>
    <p>Descripción</p>
    <button>CTA</button>
  </div>
</section>
```

```css
.hero {
  display: grid;
  grid-template-columns: 1fr;
  grid-template-rows: 1fr;
  min-height: 500px;
}

.hero-image,
.hero-content {
  grid-column: 1;
  grid-row: 1;
  /* Ambos en misma celda = overlap */
}

.hero-image {
  object-fit: cover;
  width: 100%;
  height: 100%;
  filter: brightness(0.7);
}

.hero-content {
  z-index: 1;
  display: flex;
  flex-direction: column;
  justify-content: center;
  align-items: center;
  padding: 48px;
  color: white;
}
```

### Flexbox vs. Grid: Guía de Decisión

| Situación | Usar |
|-----------|------|
| Navegación horizontal | **Flexbox** |
| Centrar elemento | **Flexbox** |
| Distribuir espacio entre items | **Flexbox** |
| Layout 1D (solo filas O columnas) | **Flexbox** |
| Layout completo de página | **Grid** |
| Gallery de imágenes | **Grid** |
| Layout 2D (filas Y columnas) | **Grid** |
| Overlapping elements | **Grid** |
| Alineación precisa en 2 ejes | **Grid** |

### Combinar Flexbox y Grid

En proyectos reales, frecuentemente usas ambos:

```html
<div class="page-grid">
  <!-- Grid para layout general -->
  <header class="header">
    <!-- Flexbox para navegación -->
    <nav class="nav">
      <a href="/">Logo</a>
      <div class="nav-links">
        <a href="/cursos">Cursos</a>
        <a href="/about">About</a>
      </div>
    </nav>
  </header>
  
  <main class="main">
    <!-- Grid para cards -->
    <div class="course-grid">
      <article class="course-card">
        <!-- Flexbox dentro del card -->
        <div class="card-content">
          <h3>Título</h3>
          <p>Descripción</p>
          <button>Ver más</button>
        </div>
      </article>
    </div>
  </main>
</div>
```

## 3.2.4. Espaciado Consistente (Regla del 8pt Grid)

El sistema 8pt grid usa múltiplos de 8 para todo espaciado, creando armonía visual y simplificando decisiones de diseño.

### Por Qué 8pt

**Ventajas:**
- ✓ Divisible por 2 (escalado fácil)
- ✓ Compatible con densidades de pantalla (2x, 3x)
- ✓ Crea ritmo visual consistente
- ✓ Reduce decisiones arbitrarias
- ✓ Facilita handoff a desarrollo

### Escala de Espaciado

```css
:root {
  /* Espaciado base */
  --space-1: 8px;   /* 0.5rem */
  --space-2: 16px;  /* 1rem */
  --space-3: 24px;  /* 1.5rem */
  --space-4: 32px;  /* 2rem */
  --space-5: 40px;  /* 2.5rem */
  --space-6: 48px;  /* 3rem */
  --space-8: 64px;  /* 4rem */
  --space-12: 96px; /* 6rem */
  --space-16: 128px;/* 8rem */
}
```

### Aplicación Práctica

**Componentes pequeños: 8-16px**
```css
.button {
  padding: 8px 16px;
  gap: 8px;
}

.input {
  padding: 12px 16px;
}
```

**Componentes medianos: 16-24px**
```css
.card {
  padding: 24px;
  gap: 16px;
}

.section {
  margin-bottom: 24px;
}
```

**Componentes grandes: 32-64px**
```css
.section-spacing {
  margin-bottom: 64px;
}

.hero {
  padding: 96px 24px;
}
```

### Excepciones al 8pt Grid

**Cuándo romper la regla:**
- Bordes: 1px, 2px son aceptables
- Tipografía: line-height puede ser decimal
- Iconos: tamaños comunes (16px, 20px, 24px, 32px)

## 3.2.5. Composición y Alineación

### Principios de Composición

**1. Balance**
- **Simétrico:** Igual peso visual a ambos lados
- **Asimétrico:** Diferentes pero equilibrados

**2. Contraste**
- Tamaño, color, forma
- Crea jerarquía e interés

**3. Énfasis**
- Punto focal claro
- Guía atención del usuario

**4. Ritmo**
- Repetición de elementos
- Crea cohesión visual

**5. Unidad**
- Elementos funcionan como todo coherente
- Consistencia en estilo

### Técnicas de Alineación

**Alineación a Grid:**
```css
.container {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
}

/* Todo alineado a columnas */
.element {
  grid-column: 2 / 8;
}
```

**Alineación Óptica:**

A veces alineación matemática no se ve alineada visualmente. Ajusta ópticamente:

```css
/* Logo con espacio visual desigual */
.logo {
  /* En vez de centrar matemáticamente */
  margin-left: -2px;  /* Compensar espacio visual */
}

/* Íconos play: centrar ópticamente */
.play-icon {
  transform: translateX(2px);  /* Ajuste óptico */
}
```

### Reglas de Oro

1. **Menos es más:** Espacios generosos mejoran legibilidad
2. **Consistencia:** Usar mismo sistema en todo el sitio
3. **Jerarquía clara:** Usuario debe saber qué es importante
4. **Alineación:** Todo debe alinearse a algo
5. **Respiración:** Dar espacio al contenido

## Conclusiones Clave

- **Patrones de lectura:** F-Pattern, Z-Pattern y Layer Cake guían organización de contenido
- **Sistema de grillas:** 12 columnas ofrece máxima flexibilidad
- **Flexbox vs Grid:** Flexbox para 1D, Grid para 2D, combinados en proyectos reales
- **8pt grid:** Sistema de espaciado consistente y escalable
- **Composición:** Balance, contraste, énfasis crean diseños efectivos
- **Responsive:** Layouts deben adaptarse a diferentes dispositivos
