# **2.7. Accesibilidad e Inclusión**

Diseñar accesiblemente no es solo cumplir regulaciones; es asegurar que todas las personas, independientemente de sus capacidades, puedan usar el producto.

## **2.7.1. Principios WCAG (Perceptible, Operable, Comprensible, Robusto)**

**Web Content Accessibility Guidelines (WCAG 2.1):**

Estándar internacional para accesibilidad web, organizado en 4 principios fundamentales.

### **PRINCIPIO 1: PERCEPTIBLE**

La información y componentes de UI deben presentarse de manera que los usuarios puedan percibirlos.

**Directrices clave:**

**1.1 - Alternativas de Texto:**

```html
<!-- Imágenes informativas -->
<img src="curso-ux.jpg" alt="Estudiantes colaborando en proyecto de diseño UX">

<!-- Imágenes decorativas -->
<img src="pattern.svg" alt="" aria-hidden="true">

<!-- Íconos funcionales -->
<button aria-label="Buscar">
  <svg aria-hidden="true"><!-- icon --></svg>
</button>
```

**1.3 - Adaptable:**

```html
<!-- ✓ Estructura semántica -->
<article>
  <header>
    <h1>Introducción al Diseño UX</h1>
  </header>
  <section>
    <h2>Descripción</h2>
  </section>
</article>
```

**1.4 - Distinguible:**

```css
/* Contraste suficiente */
.text-primary { color: #1A1A1A; /* Ratio 16:1 */ }

/* No usar solo color */
.required-field label::after {
  content: " *";
  color: red; /* + asterisco visible */
}
```

### **PRINCIPIO 2: OPERABLE**

Componentes de UI y navegación deben ser operables.

**2.1 - Accesible por Teclado:**

```html
<nav>
  <a href="#main" class="skip-link">Saltar al contenido</a>
</nav>
```

```javascript
function handleKeyDown(event) {
  if (event.key === 'Enter' || event.key === ' ') {
    event.preventDefault();
    handleClick();
  }
}
```

**2.4 - Navegable:**

```html
<!-- Landmarks ARIA -->
<header role="banner">...</header>
<nav role="navigation">...</nav>
<main role="main" id="main-content">...</main>
<footer role="contentinfo">...</footer>

<!-- Breadcrumbs -->
<nav aria-label="Breadcrumb">
  <ol>
    <li><a href="/">Inicio</a></li>
    <li aria-current="page">UX Design</li>
  </ol>
</nav>
```

### **PRINCIPIO 3: COMPRENSIBLE**

Información y operación de UI deben ser comprensibles.

**3.1 - Legible:**

```html
<html lang="es">
<p>El término <span lang="en">responsive design</span>...</p>
```

**3.3 - Asistencia de Input:**

```html
<label for="email">
  Correo electrónico
  <span class="required" aria-label="requerido">*</span>
</label>
<input 
  type="email" 
  id="email"
  aria-required="true"
  aria-describedby="email-help"
>
<span id="email-help">Usaremos tu email para actualizaciones</span>

<!-- Errores claros -->
<div class="error" role="alert">
  <strong>Error:</strong> El email debe incluir un @
</div>
```

### **PRINCIPIO 4: ROBUSTO**

Contenido debe ser interpretado por variedad de user agents, incluyendo tecnologías asistivas.

```html
<!-- HTML válido, IDs únicos, ARIA correcto -->
<button 
  aria-expanded="false"
  aria-controls="menu-dropdown"
>
  Menú
</button>
<ul id="menu-dropdown" hidden>
  <li><a href="#">Opción 1</a></li>
</ul>
```

## **2.7.2. Niveles de Conformidad (A, AA, AAA)**

### **Nivel A (Mínimo):**

Requisitos básicos.

Ejemplos:
- Alt text en imágenes
- Funcionalidad por teclado
- Evitar solo color para información

### **Nivel AA (Recomendado):**

Objetivo estándar. **Meta recomendada: WCAG 2.1 Nivel AA**

Ejemplos:
- Contraste 4.5:1 para texto
- Resize texto hasta 200%
- Navegación consistente
- Labels en formularios

### **Nivel AAA (Óptimo):**

Nivel más alto. Difícil para todo el contenido.

Ejemplos:
- Contraste 7:1
- Sin límites de tiempo
- Sign language para video

### **Checklist AA para EduConnect:**

```
Perceptible:
□ Alt text en imágenes informativas
□ Captions en videos
□ Contraste texto ≥ 4.5:1
□ Contraste UI ≥ 3:1
□ No solo color para información
□ Texto redimensionable 200%

Operable:
□ Funcionalidad accesible por teclado
□ Focus visible
□ Múltiples formas de encontrar páginas
□ Headings descriptivos
□ Focus order lógico

Comprensible:
□ Idioma de página definido (lang="es")
□ Navegación consistente
□ Labels en form inputs
□ Errores identificados con sugerencias

Robusto:
□ HTML válido
□ ARIA usado correctamente
□ Status messages identificables
```

## **2.7.3. Diseño para Navegación por Teclado**

### **Fundamentos:**

1. **Todo debe ser accesible vía teclado**
2. **Orden lógico de tabulación**
3. **Focus visible**
4. **No keyboard traps**

### **Teclas Estándar:**

- **Tab:** Navegar adelante
- **Shift + Tab:** Navegar atrás
- **Enter:** Activar links/buttons
- **Space:** Activar buttons, checkboxes
- **Arrow keys:** Navegación dentro componentes
- **Esc:** Cerrar modals/menús

### **Focus Management:**

```css
/* ✓ Focus visible */
:focus-visible {
  outline: 3px solid var(--blue-500);
  outline-offset: 2px;
}

/* NUNCA sin alternativa */
/* ✗ MALO */
*:focus { outline: none; }
```

### **Skip Links:**

```html
<a href="#main-content" class="skip-link">
  Saltar al contenido principal
</a>

<style>
.skip-link {
  position: absolute;
  top: -40px;
  left: 0;
  background: var(--blue-600);
  color: white;
  padding: 8px;
  z-index: 100;
}

.skip-link:focus {
  top: 0;
}
</style>
```

### **Modales Accesibles:**

```javascript
function openModal(modalId) {
  const modal = document.getElementById(modalId);
  const lastFocusedElement = document.activeElement;
  
  modal.hidden = false;
  modal.setAttribute('aria-hidden', 'false');
  
  // Focus primer elemento
  const firstFocusable = modal.querySelector('button, [href], input');
  firstFocusable.focus();
  
  // Trap focus
  trapFocus(modal);
  
  // Guardar para restaurar
  modal.dataset.returnFocus = lastFocusedElement;
}

function closeModal(modalId) {
  const modal = document.getElementById(modalId);
  modal.hidden = true;
  
  // Restaurar focus
  const returnElement = document.querySelector(...);
  returnElement.focus();
}
```

## **2.7.4. Diseño Inclusivo y Diversidad**

Diseño inclusivo considera diversidad de usuarios en todas sus dimensiones.

### **Dimensiones de Diversidad:**

**1. Capacidades Físicas:**

Consideraciones:
- Botones ≥ 44x44px touch
- No gestos complejos
- Tiempo suficiente

**2. Capacidades Visuales:**

Consideraciones:
- Screen reader support
- Contraste suficiente
- No solo color
- Modo oscuro disponible

**3. Capacidades Auditivas:**

Consideraciones:
- Captions en videos
- Transcripciones
- Alternativas visuales a alertas sonoras

**4. Capacidades Cognitivas:**

Consideraciones:
- Lenguaje claro y simple
- Instrucciones paso a paso
- Evitar distracciones
- Consistencia en UI
- Opciones de personalización

**5. Diversidad Cultural:**

Consideraciones:
- Internacionalización (i18n)
- Formatos locales (fecha/hora)
- Sensibilidad cultural
- RTL support (árabe)

**6. Diversidad Demográfica:**

Consideraciones:
- Representación visual diversa
- Lenguaje inclusivo
- No asumir expertise

### **Principios de Diseño Inclusivo:**

**1. Equitativo:**

```
✗ Solo login con facial recognition
✓ Múltiples opciones de autenticación
```

**2. Flexible:**

```
✓ Tamaño de texto ajustable
✓ Modo oscuro/claro
✓ Atajos configurables
✓ Velocidad de animación ajustable
```

**3. Simple e Intuitivo:**

```
✓ Iconos + labels
✓ Lenguaje plano
✓ Flujos lineales
```

**4. Información Perceptible:**

```
✓ Contraste suficiente
✓ Texto grande disponible
✓ Feedback multi-sensorial
```

**5. Tolerante a Errores:**

```
✓ Confirmación en destructivas
✓ Undo disponible
✓ Validación antes de submit
✓ Auto-save
```

### **Implementación en EduConnect:**

```html
<!-- Representación diversa -->
<img src="instructor-1.jpg" alt="Instructor latinoamericano...">

<!-- Lenguaje inclusivo -->
<h2>Te damos la bienvenida</h2>

<!-- Personalización -->
<section class="preferences">
  <h3>Preferencias de Accesibilidad</h3>
  <label>
    <input type="checkbox" id="reduce-motion">
    Reducir animaciones
  </label>
  <label>
    <input type="checkbox" id="high-contrast">
    Alto contraste
  </label>
</section>
```

```css
/* Respetar preferencias */
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

@media (prefers-color-scheme: dark) {
  :root {
    --color-bg: var(--gray-900);
    --color-text: var(--gray-50);
  }
}
```

## **2.7.5. Herramientas de Evaluación de Accesibilidad**

### **Herramientas Automatizadas:**

**1. axe DevTools**
- Chrome/Firefox extension
- Escaneo automático
- Identifica problemas WCAG

**2. WAVE (WebAIM)**
- Evaluación visual
- Identifica errores, alerts

**3. Lighthouse (Chrome DevTools)**
- Audit de accesibilidad
- Score de 0-100

**4. Pa11y**
- CLI tool
- Automatizable en CI/CD

### **Herramientas Manuales:**

**1. Screen Readers:**
- **NVDA (Windows):** Gratuito
- **JAWS (Windows):** Profesional
- **VoiceOver (macOS/iOS):** Integrado
- **TalkBack (Android):** Integrado

**2. Simuladores de Daltonismo:**
- Color Oracle
- Chrome DevTools > Rendering
- Figma: Stark plugin

**3. Contrast Checkers:**
- WebAIM Contrast Checker
- Contrast Ratio (Lea Verou)
- Colour Contrast Analyser

### **Proceso de Testing:**

**1. Automatizado:**

```bash
lighthouse https://educonnect.com --view
axe https://educonnect.com
pa11y https://educonnect.com
```

**2. Manual Keyboard:**

```
□ Desconectar mouse
□ Navegar solo con Tab
□ Verificar focus visible
□ Activar funciones con Enter/Space
□ Probar Esc en modales
□ Verificar no keyboard traps
```

**3. Screen Reader:**

```
□ Activar NVDA/VoiceOver
□ Escuchar página completa
□ Verificar alt text
□ Probar navegación por landmarks
□ Verificar formularios
□ Probar estados dinámicos
```

**4. Visual:**

```
□ Zoom a 200%
□ Probar en mobile (320px)
□ Simular daltonismo
□ Verificar contraste
```
